# Data Cleaning and Exploratory Data Analysis

## Data Cleaning
As aforementioned, the dataset used in this project comes from two separate tables from Food.com: one containing recipe information and another containing user interactions such as ratings. Because ratings are generated by users reviewing specific recipes, the first step in our cleaning process was to combine these two sources so that each recipe could be analyzed alongside the ratings it received. We merged the recipes dataset with the interactions dataset using the recipe identifier (id). This left merge ensured that each recipe was associated with all ratings that users submitted for it.

Next, we addressed an issue with the rating values. In the interactions dataset, some ratings were recorded as 0. According to the project documentation, a rating of 0 does not represent a true user evaluation but instead indicates missing data. Because the rating system on Food.com ranges from 1 to 5 stars, keeping these zeros would artificially lower average ratings. To prevent this distortion, we replaced all ratings equal to 0 with missing values (NaN). This allowed us to treat them properly as missing observations rather than actual ratings.

After handling missing ratings, we recomputed the average rating for each recipe. Since a recipe can receive many ratings from different users, we grouped the data by recipe ID and calculated the mean of the available ratings. Because pandas ignores missing values by default when computing means, this ensured that only valid ratings contributed to the final average rating for each recipe.

The recipe dataset also contained a nutrition column where nutritional information was stored as a single string containing a list of values. This format made it difficult to analyze specific nutrients directly. To make the data usable for analysis, we converted this string representation into a structured format by splitting it into separate columns representing calories, total fat, sugar, sodium, protein, saturated fat, and carbohydrates. This allowed us to examine specific nutritional characteristics of each recipe rather than working with a single combined field.

To support our analysis comparing different types of recipes, we then created several binary indicator variables based on whether a recipe’s nutritional value was above or below the median of the dataset. For example, recipes with sugar content above the median were labeled as “high-sugar,” while those below the median were considered “low-sugar.” Similar indicators were created for calories, protein, carbohydrates, and fat. Using the median as a threshold allowed us to divide recipes into roughly balanced groups while avoiding the influence of extreme values.

Finally, we removed any rows that were missing values in key variables required for our analysis, such as the computed average rating or the nutritional indicator variables. Since our central question compares ratings across nutritional categories, observations missing these fields could not contribute meaningfully to the analysis. Removing them ensured that the comparisons we perform are based on complete and interpretable data.

Together, these cleaning steps transformed the raw Food.com data into a structured dataset where each recipe includes a reliable average rating and clearly defined nutritional attributes.
